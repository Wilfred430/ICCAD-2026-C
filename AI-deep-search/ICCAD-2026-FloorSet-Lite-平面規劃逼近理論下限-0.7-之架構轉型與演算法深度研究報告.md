# ICCAD 2026 FloorSet-Lite 平面規劃逼近理論下限 0.7 之架構轉型與演算法深度研究報告

## 理論極限之數學拆解與系統性瓶頸分析
針對 ICCAD 2026 CAD Contest Problem C（FloorSet-Lite）之平面規劃挑戰，將成本函數（Cost）逼近理論下限 $0.7$ 的條件極度嚴苛。從競賽評分公式 $Cost = (1 + 0.5 \cdot (\max(0, HPWL\_gap) + \max(0, Area\_gap))) \cdot \exp(2 \cdot V_{rel}) \cdot \max(0.7, RT^{0.3})$ 可推導出，要達到極限分數，演算法必須同時滿足四個維度的絕對完美：第一，絕對的幾何合法性（零重疊與完全符合固定形狀）；第二，面積與線長表現必須完全追平甚至超越基準最佳解（使 Gap 降至 $0$）；第三，針對軟約束（Boundary、Grouping、MIB）的違規率 $V_{rel}$ 必須為絕對的 $0$；第四，運行時間必須小於全場中位數的 $30\%$，以觸發 $0.7$ 的底層截斷下限。
在現有的兩大技術路線中，離散拓樸搜尋（路線 A）結合 B*-tree 與 Contour Packer 的組合，雖然在合法性上具備建構式保證（By-construction），但其實際面積差距（Area_gap）存在 $0.28$ 至 $0.40$ 的結構性下限。此現象源於 Contour Packer 的左下貼齊規則無法表徵高密度最佳解中如拼圖般互相咬合（Jigsaw-like）的拓樸結構。另一方面，連續解析式佈局（路線 C）雖在執行速度與線長優化上表現優異，但其依賴的約束圖最長路徑壓縮法（Constraint-graph longest-path compaction）缺乏對固定外框（Fixed-outline）的感知能力，導致合法化過程中佈局外溢，面積使用率僅能達到約 $0.55$ 至 $0.60$。
此外，賽制中對於 $100$ 個驗證案例採用 $e^{n/12}$ 的指數型加權平均，意味著模組數量 $n=120$ 的大型測試案例對總分的影響力，是 $n=21$ 案例的數千倍。這要求所選用的技術架構不能僅在小規模數據上收斂，而必須具備 $O(n \log n)$ 或更佳的運算複雜度擴展性，且在極高維度的解空間中仍能穩定收斂至零違規。本報告基於 2023 至 2026 年間最新發表的電子設計自動化（EDA）與生成式人工智慧文獻，針對上述瓶頸，提出五個維度的具體突破架構與演算法重構方案。

## 零重疊與高密度兼具之非輪廓打包與合法化方法
要突破 B*-tree 結合 Contour Packer 所造成的結構性面積下限，必須放棄依賴單一增長方向的輪廓線技術，轉向能夠同時處理二維擠壓與精確空間分配的數學規劃模型。近年來的研究指出，高密度平面規劃的本質是在維持拓樸相對關係的前提下，進行死角（Whitespace）的絕對最小化。
在消除白縫與重疊的領域中，*JigsawPlanner* (ICCAD 2024) 提出了一種針對複雜模組消除白縫的拼圖式平面規劃器。該研究證實，若要達成近乎完美的面積利用率，模組在打包過程中必須具備非單調的二維滑動自由度。與此同時，*Flora* (2025) 提出了一種具備零白縫（Zero-whitespace）特性的三階段平面規劃演算法。Flora 在局部塑形與模組邊界精煉階段，透過線性時間的空隙重新分配機制，徹底消除了傳統拓樸打包後產生的結構性死角。這代表了在維持幾何合法性的同時，達到 GT（Ground Truth）利用率 $0.965$ 是完全可行的。
為了在不依賴迭代式「合法化並檢查（Legalize-then-check）」的前提下達成此目標，最佳的實作路徑是採用基於約束圖結合線性規劃（Constraint-Graph with Linear Programming, CG-LP）的打包器。當預測模型或搜尋演算法輸出 Sequence Pair 或拓樸圖時，打包器不再依賴貪婪的左下靠攏，而是將拓樸轉化為水平約束圖（HCG）與垂直約束圖（VCG）。接著，將佈局外框作為硬性約束寫入線性規劃模型中。現代的單純形法（Simplex Method）或內點法（Interior Point Method）求解器在處理百量級變數時，其運算時間遠低於一毫秒，能夠在確保絕對零重疊的同時，自動將所有模組壓縮至最緊密的狀態，達到「合法性免費」與「緊密度免費」的雙重效益。

## 固定外框導向之高密度連續場收斂與合法化
路線 C（解析式連續佈局）目前在執行時間上佔有絕對優勢，但其 eDensity 密度場與後端合法化器之間存在嚴重的銜接斷層。現有的最長路徑壓縮法（Longest-path compaction）是一種無邊界感知的推擠演算法，當模組發生重疊時，該演算法會無限制地將模組向外推移，導致原本在固定外框內已經高度收斂的佈局瞬間崩潰，進而引發面積超標的致命懲罰。要解決此問題，合法化器必須將固定外框視為最高優先級的絕對屏障。
*IncreMacro* (ISPD 2024 / TCAD 2025) 提出了一種專為高密度混合尺寸設計的漸進式合法化框架。該框架針對解析式佈局的結果，設計了基於約束圖的線性規劃合法化器（Constraint-graph-based LP for legalization）。與傳統無邊界推擠不同，IncreMacro 在 LP 公式中強制加入了 $0 \le x_i \le W_{max} - width_i$ 與 $0 \le y_i \le H_{max} - height_i$ 的嚴格邊界約束。當消除重疊所需的空間大於局部可用白縫時，這種公式化會迫使壓力向內部未完全壓縮的區域傳導，而不是向外框外溢，從而在保證零重疊的前提下完美維持外框限制。
另一方面，要減少合法化器的負擔，前端的密度場優化也必須進行根本性的改良。*CSF (Conjugate Subgradient Floorplanning)* (2025) 提出了一種結合 Q-Learning 的共軛次梯度演算法，專門處理固定外框平面規劃中的非平滑（Nonsmooth）目標函數。傳統的 eDensity 方法依賴平滑近似（如 Poisson 方程式的頻域解），這在固定外框邊緣容易產生梯度失真。CSF 則直接將重疊面積與超出邊界寬度（Out-of-bound width）設定為非平滑懲罰項，並利用次梯度（Subgradient）進行精確下降。結合 Q-Learning 自適應調整步長，CSF 能夠在探索與利用中取得極佳平衡，使得模組在進入合法化階段前，即已高度集中於合法解的流形附近，大幅降低了後續修復導致密度崩潰的風險。

## 具備合法性約束注入之連續座標擴散與生成模型
在生成式 AI 應用於平面規劃的領域，傳統的獨立座標回歸模型（如直接預測 $x, y$ 座標）已被證明存在嚴重的數學天花板：當存在多個對稱且合法的佈局解時，均方誤差（MSE）或平滑 L1 損失函數會驅使模型學習這些合法解的平均值，而這個平均值通常會導致模組全數堆疊於晶片中央，形成不可行的解（模式崩塌）。為突破此限制，必須採用具備機率採樣能力且能將物理約束注入生成過程的架構。
*DiffPlace* (2025) 提出了一種超越循序生成的條件去噪擴散框架（Conditional Denoising Diffusion Framework）。不同於 MaskPlace 或 GraphPlace 將平面規劃視為逐步放置的馬可夫決策過程（這會導致嚴重的錯誤累積與極長的推論時間），DiffPlace 同時對所有模組的位置進行去噪優化。其最核心的突破在於「解耦引導機制（Decoupled guidance mechanism）」：模型在採樣過程中，不僅利用能量條件化（Energy-based Conditioning）來優化線長與擁塞度，更透過顯式的「流形梯度注入（Explicit manifold gradient injection）」主動消除局部物理約束的違規。這意味著零重疊與邊界約束被轉化為引導採樣方向的流形力場，確保擴散模型輸出的連續座標天生具備極高的合法性，從根本上解決了生成座標不可行的痛點。
然而，標準擴散模型在推論時需要數十步的反覆運算，難以在 $1-3$ 秒的嚴格時間預算內完成。為了解決速度問題，*LayoutFlow* (ECCV 2024) 引入了流匹配（Flow Matching）技術應用於佈局生成。與擴散模型那種充滿雜訊、不斷改變方向的軌跡不同，流匹配學習的是一條從高斯基礎分佈指向目標資料分佈的平滑向量場（Smooth and directed paths）。這種平滑的幾何特性使其極為適合搭配一致性蒸餾（Consistency Distillation）技術，將原本需要多步的生成過程壓縮至 $1$ 到 $4$ 步之內。透過 LayoutFlow 框架，可以在不到一百毫秒的時間內，生成具備高度合法性且逼近 GT 密度的初始連續座標，完美配合後端的 CG-LP 合法化器。此外，*AutoFloorplan* (ICLR 2026) 提出的文字梯度引導修復機制（Textual Gradient-Guided Repair），雖主要用於啟發式演算法的程式碼演化，但其「透過錯誤反饋直接修正架構」的核心思想，亦可作為我們設計神經網路架構損失函數的理論指引。

## 保證零違規的柔性剛體與約束優化演算法
在評分公式中，$V_{rel}$ 的存在是一個乘法地獄：只要發生一次軟約束（Grouping、MIB、Boundary）違規，$Cost$ 就會被乘上大於 $1$ 的指數懲罰，直接宣告無法逼近 $0.7$ 的理論下限。目前的 S1 解法將同群組模組壓縮為單一絕對剛體（Super-block），這種作法過度限縮了解空間，導致剛體在固定外框的高密度環境中無法靈活穿插，最終被成本排名機制拒絕。
要同時保證約束滿足與高密度，必須將約束處理從「絕對剛性綁定」轉變為「柔性數學限制」。*RulePlanner* (ICML 2026) 針對三維平面規劃中錯綜複雜的設計規則，提出了一套整合所有約束的強化學習框架。該框架的突破點在於徹底捨棄事後修復，而是透過多種遮罩矩陣（Matrix Representations）直接重塑行動空間（Action Space）。
具體而言，RulePlanner 引入了「相鄰模組遮罩（Adjacent Block Mask）」與「相鄰終端遮罩（Adjacent Terminal Mask）」。對於 Grouping 約束，系統動態生成一張表示合法鄰接區域的二維矩陣（Availability Mask）；任何試圖將群組成員放置於此遮罩外的優化步驟都會在底層被強制過濾。將此思維移植到我們的解析式路線（路線 C）或 CG-LP 合法化器中，意味著我們不應在幾何上將模組預先合併，而是將群組約束轉化為優化器內部的**等式與不等式系統**。
例如，當模組 A 與模組 B 屬於同一 Grouping 群組時，我們不在佈局前將其黏合，而是在線性規劃或梯度下降的約束條件中，加入 $\min(\vert{}x_A + width_A - x_B\vert{}, \vert{}x_B + width_B - x_A\vert{}) = 0$ 且兩者在 Y 軸上必須有重疊的條件。這創造了所謂的「柔性剛體（Flexible Rigid Body）」：群組內的模組可以像軌道列車一樣沿著彼此的邊界自由滑動（Sliding），甚至允許其他非群組模組在不切斷它們連線的前提下填補周圍的白縫。同理，對於 MIB（多重例現區塊）約束，我們將共享相同形狀的模組的長寬變數在優化圖中綁定為同一個變數節點（Node Sharing），但其 $x, y$ 座標維持獨立。這種約束表徵方法不僅能在數學結構上達成 $100\%$ 的 $V_{rel}=0$ 保證，同時釋放了極大的幾何自由度以逼近 GT 的極高密度。

## 突破框架稅：CPU 毫秒級推論之極致加速技術
在 ICCAD 2026 Problem C 的賽制中，運行時間（RT）的獎勵下限被鎖死在全場中位數的 $30\%$，亦即 $RT \le 0.305$ 才能使得 $RT^{0.3}$ 撞到 $0.7$ 的地板。這代表即便演算法的 Big-O 複雜度極低，任何來自程式語言或深度學習框架的啟動延遲（Startup Latency）都將成為致命傷。
針對邊緣運算與即時音訊處理領域的神經網路推論延遲研究 提供了極具價值的數據基準。研究指出，在純 CPU 環境下，載入完整的 PyTorch 框架並進行首次推論（Time-to-first-prediction）大約需要消耗 $1005$ 毫秒。對於小型測試案例（如 $n=21$），這 $1$ 秒鐘的「框架稅（Framework Tax）」可能就佔據了總運算時間的 $80\%$ 以上，直接導致 RT 懲罰暴增，徹底摧毀逼近 $0.7$ 的可能性。
為了將推論時間壓縮至次秒級甚至毫秒級，必須徹底剝離 Python 執行環境，轉向原生的底層編譯技術。
首先，最直接且高投報率的手段是採用 **ONNX Runtime C++ API**。文獻指出，將神經網路模型導出為 ONNX 格式，並透過 C++ 原生環境載入 ONNX Runtime，可將首次推論延遲大幅縮減至約 $237$ 毫秒。這是因為 ONNX Runtime 被設計為純粹的推論邊界，免除了神經網路建構、梯度追蹤初始化以及 Python 解譯器啟動的龐大開銷。
若要追求極致的效能，則應考慮**無框架原生靜態編譯（Native Static Compilation）**。如同 OpenNN 或現代的 ggml 架構，若預測模型（如 Graph-Transformer 或 LayoutFlow）的結構相對單純（僅包含標準的卷積、注意力機制與多層感知機），可以將模型權重直接轉換為 C++ 標頭檔，並將張量運算直接對接至 BLAS 函式庫（如 OpenBLAS 或 Intel MKL）。基準測試證實，這種原生執行檔的啟動至預測時間僅需約 $36$ 毫秒。結合多執行緒 C++ 實作，這種架構能確保即使在處理 $100$ 個驗證案例的迭代迴圈中，推論階段的時間消耗也幾乎可以忽略不計，為複雜的連續優化或 LP 合法化器保留充足的運算預算。
